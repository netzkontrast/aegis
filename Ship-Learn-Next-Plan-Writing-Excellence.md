## üí° Key Principles

**1. Clarity over cleverness**
Simple, direct language beats elaborate prose. Respect the reader's time.

**2. Front-load key information**
Put the most important point first. Let readers decide if they need details.

**3. Show don't tell**
Use concrete examples. Avoid abstract explanations.

**4. Active voice, specific verbs**
"Create the document" beats "The document should be created."

**5. One idea per unit**
One paragraph = one idea. One section = one concept. Atomic thinking.

**6. Connect generously**
Link to related concepts. Build the knowledge graph.

**7. Write for the reader, not yourself**
What do THEY need to know? What questions will THEY have?
